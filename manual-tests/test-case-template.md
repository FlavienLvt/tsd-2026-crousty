# Test Case Template

This is the standard format our team uses for all manual test cases.

| Field | Description |
| :--- | :--- |
| **Test Case ID** | Unique identifier (e.g., TC-001) |
| **Title** | Short descriptive name |
| **Feature** | Tested functionality (e.g., Login, Cart) |
| **Priority** | High / Medium / Low |
| **Type** | Functional / UI / Negative |
| **Preconditions** | What must be true before execution |
| **Test Data** | Inputs, user accounts, etc. |
| **Steps** | Numbered, step-by-step actions |
| **Expected Result**| Observable expected behavior |
| **Actual Result** | Filled during execution |
| **Status** | Not Run / Passed / Failed |
| **Automation** | Yes / No (Candidate for UI automation) |
